HTML

  • HTML stands for Hypertext Markup Language
  • HTML is used to give a website's content structure.
  • HTML is made up of the elements that work together to provide structure
  • HTML elements ARE case-sensitive. Declarations are NOT case-sensitive.
  • Two important elements when using HTML are: the head element and the body element
  • Some of the elements are visibile to the user while others are not
  • Element types often used to add content to pages: Text, Image, Header, and List
  • Element types often used to divide and describe a page: Meta-related, Content division, and Semantic
  • The head element contains information about the webpage.
  • The information that the head element contains is called metadata
  • The head element's metadata/information assists the browser to render the page correctly
  • Elements within the head are NOT visible to the user
  • The title tag goes inside the head element
  • The body element represents the visible content shown to the user.
  • An element refers to the entire element from opening tag to closing tag.
  • A tag refers only to what appears inside the angle brackets.
  • IMPORTANT: HTML attributes are ALWAYS defined in the beginning element, NEVER the closing element.

CSS

  • CSS means Cascading Style Sheets
  • Three ways to style a webpage using CSS: inline, internal style sheet, and external style sheet.
  • Use inline CSS to style an element directly in the HTML file
  • The most common way to style a webpage is by using an external CSS style sheet.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The Box Model refers to the idea that HTML elements are esstemtially boxes that nest inside each other.
  • A HTML element box consists of margins outside the element, and the element's borders, padding, and content.

Git

  • git status: checks what branch we are currently on
  • git checkout -b example/branch-name: creates a new branch and switches to it
  • git fetch: Download objects and refs from another repository
  • git pull: Fetch from and integrate with another repository or a local branch
  • git push: Update remote refs along with associated objects
  • git help -a and git help -g: list available subcommands and some concept guides.
  • git help command or git help concept: to read about a specific subcommand or concept.
  • git help git: for an overview of the system.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • placing JavaScript near the bottom of an HTML page is one way to allow the browser to interpret code
  • Document Object Model (DOM) API, which has the capability to manipulate documents.
  • Application Programming Interfaces (APIs) built into web browsers, providing functionality
  • To signify that the value is a string, enclose it in single or double quote marks.
  • Numbers don't need quotation marks
  • Boolean is a True/False value. The words true and false are special keywords that don't need quote marks.
  • Array is a structure that allows you to store multiple values in a single reference.
  • Everything in JavaScript is an object and can be stored in a variable
  • /*This is a comment with linebreaks*/ and //This is a comment without linebreaks
  • JSON (JavaScript Object Notation) is a lightweight data-interchange format, derived from JavaScript, but used by many programming languages.
  • JSON builds universal data structures that can be transferred between different environments and even across languages.
